# RepoSync 首版实施方案（含递归子模块与本地缓存）

## Summary
构建一个单机部署的 Web 管理台，用于管理多个“源仓库 -> 目标仓库”的 Git 镜像同步任务。  
首版面向单管理员使用，优先支持 GitHub 和 Gogs，支持 SSH Key 与 HTTPS Token，支持 Webhook + 定时触发，支持递归同步子模块仓库，并通过本地裸仓库镜像缓存降低重复同步耗时。系统默认安全优先，异常时阻止覆盖并记录告警。

## Key Changes
### 1. 系统形态
- 后端采用 Go，负责 API、调度、Webhook、同步执行、子模块递归、缓存管理和日志记录。
- 前端提供任务管理、凭证管理、执行日志看板和缓存管理入口。
- 单进程部署，使用本地数据库和本地缓存目录。

### 2. 核心同步能力
- 支持多个同步任务，每个任务至少包含：
  - 任务名称
  - 源仓库信息
  - 目标仓库信息
  - 凭证类型与凭证引用
  - 触发方式配置
  - 是否启用
  - 是否启用递归子模块同步
- 主仓库同步采用完整镜像语义。
- 新增递归子模块同步：
  - 解析 `.gitmodules`
  - 发现子模块仓库与引用 commit
  - 递归同步子模块仓库本身
  - 自动按目录结构生成子模块目标仓库地址
- 同步执行需保证：
  - 主仓库引用到的子模块 commit 在目标端可访问
  - 检测循环依赖和重复仓库，避免无限递归
  - 任一子模块失败时，主任务整体失败并标记失败链路
- 默认不自动 force push，目标异常时停止并告警。

### 3. 本地缓存设计
- 每个源仓库维护一个本地 bare/mirror 缓存仓库，主仓库和子模块都复用该缓存。
- 同步流程改为：
  - 先定位或创建本地镜像缓存
  - 对缓存执行增量 fetch
  - 基于缓存直接向目标仓库 push --mirror 或等效镜像推送
- 缓存键以“规范化后的源仓库 URL + 认证上下文”确定，避免不同源混用。
- 递归子模块同步时，子模块也独立命中自己的缓存，不重复全量拉取。
- 缓存默认长期保留，不自动删除；管理台提供手动清理入口。
- 需要记录缓存元数据：
  - 缓存路径
  - 最近 fetch 时间
  - 最近使用时间
  - 命中次数
  - 当前大小
- 同一仓库缓存需加锁，避免并发 fetch/push 导致损坏。

### 4. 管理台页面
- 任务列表页展示：
  - 主任务状态
  - 最近执行结果
  - 是否启用子模块递归
  - 最近一次递归同步的仓库数量
- 任务配置页新增：
  - 递归子模块开关
  - 子模块目标映射预览
- 日志/详情页展示：
  - 主仓库与子模块执行树
  - 每个节点是否命中本地缓存
  - fetch/push 耗时和错误信息
- 缓存管理页或任务详情补充展示：
  - 本地缓存状态
  - 占用空间
  - 最近更新时间
  - 手动清理操作
- 凭证管理页继续只显示脱敏信息。

### 5. 数据与接口
- 主要实体：
  - `SyncTask`
  - `Credential`
  - `SyncExecution`
  - `SyncExecutionNode`
  - `RepoCache`
- 管理 API 至少包括：
  - 任务 CRUD
  - 手动触发任务
  - 查询执行历史与递归节点详情
  - 凭证 CRUD
  - 缓存状态查询与清理
  - GitHub/Gogs webhook 接收入口
- 执行记录需额外保存：
  - 递归深度
  - 父子仓库关系
  - 源/目标仓库映射结果
  - 缓存命中情况
  - 失败节点位置

## Test Plan
- 首次同步时能创建本地裸仓库缓存并完成同步。
- 二次同步时能命中缓存，执行时间明显短于首次全量拉取。
- 主仓库与多层子模块都能各自复用本地缓存。
- 同一仓库被多个任务引用时，缓存不会被并发写坏。
- 子模块递归同步、缓存命中、目标推送三者组合场景正常。
- 缓存损坏时，系统能识别并重建，任务日志可见原因。
- 手动清理缓存后，下次同步能自动重新建立缓存。
- GitHub webhook、Gogs webhook 触发后，带缓存的同步任务可正常执行。
- SSH/HTTPS 两种认证都能覆盖主仓库与子模块访问。

## Assumptions
- 首版仍为单管理员系统，不做多用户和 RBAC。
- 首版平台适配优先 GitHub + Gogs。
- 子模块能力定义为“递归同步子模块仓库本身”。
- 子模块目标地址按目录结构自动映射生成。
- 本地缓存采用长期保留的裸仓库镜像，不做自动过期淘汰。
- 若子模块目标仓库缺失，默认按失败处理，而不是自动创建。
