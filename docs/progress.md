# RepoSync 开发进度

更新时间：2026-03-22

## 已完成

### Phase 0
- 完成仓库初始化
- 完成前后端工程骨架
- 完成系统架构、数据模型、API、roadmap 文档

### Phase 1
- 已完成 SQLite 数据层
- 已完成 `SyncTask`、`Credential`、`SyncExecution`、`SyncExecutionNode`、`RepoCache` 的基础持久化
- 已完成任务 CRUD、凭证 CRUD、缓存查询、执行历史查询 API
- 已完成前端基础管理台页面

### Phase 2
- 已接入真实 Git mirror 流程
- 已支持 `git clone --mirror`
- 已支持缓存仓库上的 `git fetch --prune origin +refs/*:refs/*`
- 已支持目标仓库上的 `git push --mirror`
- 已通过本地集成测试验证镜像所有分支与标签
- 已支持缓存命中记录与第二次执行复用缓存
- 已支持按任务指定缓存保存路径

## 进行中
- Phase 3 自动建仓
- Phase 4 递归子模块同步
- Phase 5 定时调度与 Webhook
- Phase 6 管理台完善

### Phase 3 当前进度
- 已接入 SCM Provider 抽象
- 已实现 GitHub Provider 基础流程
- 已实现 Gogs Provider 基础流程
- 已支持本地 bare 目标仓库缺失时自动初始化
- 已接入目标仓库存在性检查
- 已接入“缺失即自动建仓”的执行前置步骤
- 已将自动建仓结果写入执行节点与执行汇总
- 已通过 Provider 单元测试
- 已覆盖“检查时不存在、创建时已存在”的自动建仓竞争场景，并按幂等成功处理
- 已补充 GitHub/Gogs 自动建仓的鉴权与权限错误测试

当前限制：
- Git 凭证与 API 凭证仍共用任务层配置，尚未拆出更细粒度权限模型

### Phase 4 当前进度
- 已接入 `.gitmodules` 解析
- 已接入子模块引用 commit 读取
- 已接入主仓库到子模块的递归执行树
- 已接入按路径派生的子模块目标仓库映射
- 已接入循环依赖检测，避免无限递归
- 已通过本地集成测试验证主仓库和子模块递归镜像
- 已在主仓库推送前改写 `.gitmodules`，使目标主仓库默认引用镜像后的子模块地址
- 已通过递归 clone 集成测试验证目标主仓库可直接使用镜像子模块地址
- 已把父仓库 gitlink 改写到同步后的子模块提交，补齐多层递归子模块闭环
- 已稳定化 `.gitmodules` 改写提交，同一源分支重复同步时不会继续生成新的目标分支 SHA
- 已通过本地集成测试验证主仓库与子模块目标仓库都可在缺失时自动创建
- 已在文档中明确标注：递归子模块模式下，目标分支为可直接使用的派生提交，不再等同于源分支 commit

当前限制：
- 当前子模块认证沿用主任务目标凭证
- 标签 refs 当前保持原始镜像语义；分支 refs 会覆盖为改写过 `.gitmodules` 的提交
- 前端执行详情已能看到子模块节点，但还没有更清晰的树形可视化

### Phase 5 当前进度
- 已接入基于 cron 表达式的定时调度器
- 已在任务创建、更新、删除时同步调度注册状态
- 已提供调度状态查询接口，返回注册状态、下一次执行时间和未注册原因
- 已接入 GitHub Webhook 签名校验
- 已接入 Gogs Webhook 签名校验
- 已接入 Webhook 事件类型过滤，仅处理 push 事件
- 已接入按任务目标分支的 Webhook ref 过滤
- 已在执行期间持续写入阶段性进度日志
- 已接入 Git 命令级日志采集，记录 `clone/fetch/push` 等命令及其 stdout/stderr 输出
- 已提供执行日志 SSE 流接口
- 已记录 Webhook 的 accepted/ignored/rejected/failed 历史事件
- 已通过调度器单元测试和 Webhook 鉴权单元测试

当前限制：
- 前端尚未展示 Webhook 被忽略的具体原因

### Phase 6 当前进度
- 已重做任务页信息密度，补充调度和 Webhook 状态摘要
- 已补充独立的调度状态面板，集中查看注册状态与下一次执行时间
- 已在执行详情中补充递归执行树视图
- 已在执行详情中补充缓存命中、自动建仓、耗时和错误信息展示
- 已把任务列表中的调度、Webhook 和分支过滤状态拆成独立状态标签
- 已把执行详情树升级为可展开/收起，并支持只看失败节点
- 已补充执行详情中的触发状态面板与节点统计卡片
- 已为任务编辑区补充触发配置预览面板和 Webhook 路径预览
- 已为执行树补充节点选中高亮与独立节点详情面板
- 已补充执行日志面板，并在执行进行中自动轮询刷新
- 已将执行日志刷新升级为基于 SSE 的实时推送，保留失败时的详情回退刷新
- 已补充任务级 Webhook 历史表格，显示最近的状态、事件、ref 和结果原因
- 已为 Webhook 历史补充状态筛选和跳转到对应执行记录
- 已完成前端构建验证

当前限制：
- 执行树当前仍是轻量级可折叠视图，不是完整组件化树控件
- Webhook 历史当前支持基础状态筛选，但还没有重放、导出或聚合统计
- 当前日志实时通道基于 SSE，尚未扩展到 WebSocket 或更细粒度的节点级事件流
